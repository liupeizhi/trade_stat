# 交易统计系统需求文档

## 1. 项目概述

### 1.1 项目背景
随着股票市场的日益普及，个人投资者的交易频率和交易量不断增加。如何有效管理和分析个人交易记录，计算投资收益，评估投资策略的有效性，成为投资者面临的重要问题。传统的交易记录管理方式主要依赖券商提供的交易软件，但其功能往往有限，无法满足投资者对深度分析的需求。

### 1.2 项目意义
交易统计系统旨在为个人投资者提供一个全面、专业的交易记录管理和分析平台，帮助投资者：
- 系统化管理个人交易记录，支持多券商数据导入
- 实时掌握持仓情况和收益状况
- 深度分析交易历史，评估投资策略
- 优化投资决策，提高投资收益

### 1.3 项目目标
- 开发一个功能完整、界面友好的交易统计系统
- 支持多券商交易记录的导入和解析
- 提供全面的交易数据分析和可视化功能
- 实现交易记录的实时更新和持仓计算

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 交易记录管理
- **手动添加交易记录**：支持用户手动添加买入/卖出交易记录
- **批量上传交易记录**：支持从不同券商（海通证券、招商证券、国信证券）上传交易记录文件
- **交易记录查询**：支持按股票代码、时间范围等条件查询交易记录

#### 2.1.2 仓位管理
- **当前持仓查询**：实时查询当前持有的股票及其详细信息
- **历史清仓查询**：查询已清仓股票的交易记录和收益情况
- **持仓时间线**：展示股票持仓的历史变化

#### 2.1.3 收益分析
- **收益计算**：计算股票的实时收益和历史收益
- **收益率分析**：分析不同股票的收益率和投资效果
- **资金流分析**：分析资金的流入流出情况

#### 2.1.4 数据可视化
- **仪表盘**：展示交易概览、收益统计等关键指标
- **K线图**：展示股票的价格走势
- **成本线**：展示股票的持仓成本变化

### 2.2 页面需求

#### 2.2.1 分析仪表盘
- **页面路径**：/analysis/dashboard
- **功能**：展示交易概览、收益统计、持仓分布等关键指标
- **布局**：采用卡片式布局，包含多个统计模块

#### 2.2.2 当前持仓
- **页面路径**：/position/current
- **功能**：展示当前持有的股票列表，包括股票代码、名称、价格、数量、市值、成本、收益等信息
- **操作**：支持对持仓股票进行买入、卖出操作，查看操作记录

#### 2.2.3 历史清仓
- **页面路径**：/position/history
- **功能**：展示已清仓股票的交易记录和收益情况
- **筛选**：支持按股票代码、时间范围等条件筛选

#### 2.2.4 持仓时间线
- **页面路径**：/position/timeline
- **功能**：展示股票持仓的历史变化，包括建仓、加仓、减仓、清仓等操作

#### 2.2.5 交易详情
- **页面路径**：/trade/details
- **功能**：展示交易记录的详细信息，支持按股票代码、时间范围等条件查询

#### 2.2.6 K线图
- **页面路径**：/trade/kline
- **功能**：展示股票的K线图，支持不同时间周期的切换

#### 2.2.7 交易记录上传
- **页面路径**：/trade/upload
- **功能**：支持上传交易记录文件，选择券商类型

### 2.3 API接口需求

#### 2.3.1 仪表盘接口
- **接口路径**：/dashboard
- **方法**：GET
- **功能**：获取仪表盘数据，包括交易概览、收益统计等

#### 2.3.2 仓位管理接口
- **获取当前仓位**：/position/current_positions (GET)
- **查询历史清仓交易**：/position/stock_history_positions (POST)
- **查询某天某股票的持仓情况**：/position/day_position (GET)
- **计算区间内的所有交易日的仓位**：/position/compute (GET)

#### 2.3.3 交易记录接口
- **添加交易记录**：/trade_details (POST)
- **批量上传交易记录**：/trade_details/upload_records (POST)
- **获取某只股票的操作点**：/trade_details/trade_points (GET)
- **查询交易记录**：/trade_details/raw_query (GET)

#### 2.3.4 其他接口
- **资金流接口**：/capital_flow (GET)
- **成本线接口**：/cost_line (GET)
- **持仓天数接口**：/hold_days (GET)
- **收益接口**：/profit (GET)
- **指标接口**：/quota (GET)
- **股票信息接口**：/stockInfo (GET)
- **交易次数接口**：/trade_times (GET)
- **胜率接口**：/win (GET)

## 3. 技术架构

### 3.1 前端架构
- **框架**：React + Umi
- **UI组件**：Ant Design Pro
- **数据可视化**：ECharts
- **状态管理**：React Hooks
- **网络请求**：Umi Request

### 3.2 后端架构
- **框架**：Spring Boot
- **持久层**：MyBatis
- **数据库**：MySQL
- **API文档**：Swagger
- **事件处理**：自定义事件机制

### 3.3 数据结构

#### 3.3.1 交易记录（TradeDetail）
- **id**：交易记录ID
- **code**：股票代码
- **tradeTime**：交易时间
- **opt**：操作类型（买入/卖出）
- **price**：交易价格
- **vol**：交易量
- **commission**：手续费
- **tax**：税费

#### 3.3.2 股票持仓（StockPosition）
- **code**：股票代码
- **vol**：持仓数量
- **price**：当前价格
- **currentCost**：当前成本
- **lastClearTime**：上次清仓时间
- **buildPositionTime**：建仓时间

#### 3.3.3 每日持仓（StockPositionDay）
- **id**：记录ID
- **code**：股票代码
- **day**：日期
- **vol**：持仓数量
- **cost**：持仓成本
- **price**：收盘价格

## 4. 业务流程

### 4.1 交易记录上传流程
1. 用户选择券商类型
2. 用户上传交易记录文件
3. 系统根据券商类型选择对应的解析器
4. 解析器解析交易记录文件，提取交易数据
5. 系统保存交易记录到数据库
6. 系统触发结算事件，计算持仓和收益

### 4.2 买入/卖出流程
1. 用户选择股票和操作类型（买入/卖出）
2. 用户输入交易时间、价格、数量等信息
3. 系统验证输入信息的合法性
4. 系统保存交易记录到数据库
5. 系统触发结算事件，重新计算持仓和收益

### 4.3 持仓计算流程
1. 系统根据交易记录计算每日持仓
2. 系统计算每个交易日的持仓成本和市值
3. 系统计算收益率和盈亏情况
4. 系统更新持仓数据和收益数据

## 5. 非功能需求

### 5.1 性能需求
- **响应时间**：页面加载时间不超过3秒
- **数据处理**：支持处理至少5年的交易记录
- **并发处理**：支持至少10个用户同时在线使用

### 5.2 可靠性需求
- **数据备份**：定期备份交易数据
- **错误处理**：提供友好的错误提示和处理机制
- **系统稳定性**：系统运行稳定，无明显卡顿或崩溃

### 5.3 安全性需求
- **数据安全**：交易数据存储安全，防止未授权访问
- **输入验证**：对用户输入进行严格验证，防止恶意输入

### 5.4 可扩展性需求
- **模块化设计**：采用模块化设计，便于功能扩展
- **多券商支持**：支持扩展新的券商交易记录解析器
- **新功能集成**：便于集成新的分析工具和数据可视化组件

## 6. 部署与集成

### 6.1 部署方式
- **Docker部署**：支持Docker容器化部署
- **本地部署**：支持本地开发环境部署

### 6.2 环境要求
- **前端**：Node.js 14+
- **后端**：JDK 1.8+, Maven 3.6+
- **数据库**：MySQL 5.7+

### 6.3 集成方式
- **前后端分离**：前端和后端独立部署
- **API集成**：前端通过RESTful API与后端交互

## 7. 总结

交易统计系统是一个为个人投资者提供交易记录管理和分析的专业平台。通过系统化管理交易记录，实时计算持仓和收益，提供深度数据分析和可视化，帮助投资者更好地理解自己的投资行为，优化投资策略，提高投资收益。

系统支持多券商交易记录的导入和解析，提供全面的交易数据分析功能，包括当前持仓、历史清仓、收益分析等。前端采用现代化的React技术栈，界面友好，交互流畅；后端采用Spring Boot框架，性能稳定，扩展性强。

通过本系统，投资者可以更加科学地管理自己的投资组合，提高投资决策的准确性和效率，实现更好的投资回报。
